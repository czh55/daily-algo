题目描述。给你一个 32 位的有符号整数 x，返回将 x 中的数字部分反转后的结果。如果反转后整数超过 32 位有符号整数的范围 [−2³¹, 2³¹−1]，就返回 0。假设环境不允许存储 64 位整数。

示例。示例 1

    输入：x = 123。示例 2

    输入：x = -123。示例 3

    输入：x = 120

先模拟答题者的思考过程，从暴力解法出发，找到重复劳动，再推导优化方向。

1. 反转数字就是不断「弹出 x 的最低位、推到结果 rev 的最低位」。

2. 难点是 32 位溢出：不能先算完再判断，因为中间就可能溢出。

3. 在 rev = rev*10 + digit 之前先判断：若 rev 已经超过 INT_MAX/10，或等于且下一位过大，就必然溢出，返回 0。

4. Python 没有溢出，但仍按题意在结果超出 32 位范围时返回 0。

接下来是变量语义，先理解每个变量的定义、维护和更新，再开始编码。

变量 rev，类型 int。定义：已经反转好的部分
维护：每弹出 x 的一位就接到 rev 末尾
更新：rev = rev * 10 + digit

变量 digit，类型 int。定义：x 当前的最低位
维护：每轮取 x % 10
更新：取完后 x //= 10

落码步骤如下。

1. 取符号，转成绝对值处理（Python）；C++ 用带符号取模

2. 循环：digit = x % 10，x //= 10

3. 更新前先做溢出判断，安全后 rev = rev*10 + digit

4. 返回 rev（越界返回 0）

复杂度分析。时间复杂度 O(log|x|)，空间复杂度 O(1)。

常见坑点，请注意。

&#x2757; 溢出判断必须在 rev*10+digit 之前，否则中间结果已经溢出（题设不许用 64 位）。

&#x2757; INT_MAX 末位是 7、INT_MIN 末位是 8，边界时要单独比较最后一位。

&#x2757; 末尾有 0 会自然消失（120 ，得到 21），无需特殊处理。

必测的边界情况。

Case 1：反转后溢出

Case 2：末尾为 0
