题目描述。给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2，请你找出并返回这两个正序数组的中位数。要求算法的时间复杂度为 O(log(m+n))。

示例。示例 1

    输入：nums1 = [1,3], nums2 = [2]。示例 2

    输入：nums1 = [1,2], nums2 = [3,4]

先模拟答题者的思考过程，从暴力解法出发，找到重复劳动，再推导优化方向。

1. 暴力：合并两数组再取中位数，O(m+n)，但题目要 O(log)。

2. 中位数的本质：把两数组切成「左右两半」，左半所有数 &le; 右半所有数，且左半元素个数固定。

3. 只要确定 nums1 在哪切（切 i 个），nums2 的切法 j 就唯一确定（i + j = 左半总数）。

4. 对较短数组二分 i：若 L1 > R2 说明 i 太大，左移；若 L2 > R1 说明 i 太小，右移。

接下来是变量语义，先理解每个变量的定义、维护和更新，再开始编码。

变量 i，类型 int。定义：较短数组 nums1 划入「左半部分」的元素个数
维护：二分范围 [0, m]
更新：切分不合法时 i 左移或右移

变量 j，类型 int。定义：nums2 划入左半部分的个数，由 i 决定
维护：始终满足 i + j = 左半部分总数
更新：j = total_left - i

变量 L1,R1,L2,R2，类型 int。定义：两数组在切分处的左/右边界值（越界用 ±∞）
维护：合法切分需 L1&le;R2 且 L2&le;R1
更新：随 i,j 取值

落码步骤如下。

1. 保证在较短数组上二分（必要时交换）

2. total_left = (m + n + 1) // 2，二分 i 于 [0, m]

3. j = total_left - i，取四个边界 L1/R1/L2/R2（越界用 ±∞）

4. 若 L1 &le; R2 且 L2 &le; R1：奇数返回 max(L1,L2)，偶数返回 (max(L1,L2)+min(R1,R2))/2

5. 否则据 L1 > R2 调整二分区间

复杂度分析。时间复杂度 O(log(min(m,n)))，空间复杂度 O(1)。

常见坑点，请注意。

&#x2757; 一定在较短数组上二分，否则 j 可能为负越界。

&#x2757; 切分越界处用 ±∞ 兜底，避免访问 nums[-1] 或 nums[m]。

&#x2757; total_left 用 (m+n+1)//2，奇偶统一，奇数时中位数落在左半最大值。

必测的边界情况。

Case 1：一个数组为空

Case 2：不重叠
